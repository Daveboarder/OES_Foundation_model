Project Summary

LIBS Foundation Model

A self-supervised foundation model for LIBS. A BERT-style transformer pre-trains on physics-based Voigt spectra using bin (intensity), line-token (runtime embedding), or line-token-linear (pre-baked 14-feature HDF5 tokens + nn.Linear), then fine-tunes for classification, regression, and binned quantification.

Python ≥ 3.10 PyTorch + Lightning uv CUDA 11.8 / 12.1 / 12.4 License: MIT

2. Functionality

Pre-training (self-supervised)

  • 15 % of spectral bins are selected for masking using contiguous block sampling with variable block sizes (e.g. [25, 50, 100, 150]) and optional peak-biased selection (target fraction of masks must overlap intensity peaks).
  • BERT-style replacement at selected positions: 80% → mask token 10% → random value 10% → unchanged (see data/dataset.py — MaskedLIBSDataset).
  • The encoder predicts the original intensity at every masked bin; loss is MSE over all 15 % (not just the mask-token 80 %).
  • Mask token-embedding is applied only to type-1 positions inside SpectralEmbedding; the other selected positions keep their projected intensity vectors.

Pre-training — line-token modes

  • Requires --line_embedding_config (config/line_embedding.yaml).
  • Three HDF5 caches (built in order, all reusable): line_dict_*.h5 (theoretical dictionary), line_features_*.h5 (Voigt fits, 6 channels), line_tokens_*.h5 (14-feature tensor per line — separated tokenization).
  • scripts/build_line_tokens.py materializes line_tokens_*.h5 without training.
  • line_token — LineTokenEmbedding assembles quantum + element/ion nn.Embedding + fit features at runtime from line_features_*.h5.
  • line_token_linear — LinearLineTokenEmbedding z-scores raw 14 channels from line_tokens_*.h5 and applies nn.Linear(14, d_model).
  • Token channels include atomic number Z and ion_binary (0=I, 1=II+); fit_valid is a separate mask.
  • MIP masks valid lines and predicts max_intensity + fwhm (MaskedLineTokenDataset or MaskedLineTokensDataset).

Fine-tuning (supervised)

  • Loads encoder weights from a pretrain run (either a raw .pt state-dict or a Lightning .ckpt).
  • Four task modes supported by LIBSFinetuneModule:
    • classification — single label, CE loss
    • quantification — concentration vector, MSE (regression is a legacy alias)
    • quantification_binned — per-element 1000-way CE on discretised concentrations (mirrors upstream Daveboarder/Element_Identification)
    • both — joint classification + regression
  • Pooling of the encoder output is configurable via --pool {cls,mean,cls_mean}. The head input dimension scales accordingly (d_model or 2·d_model).
  • Separate AdamW parameter groups: encoder gets 0.1× the head LR; cosine schedule with linear warmup.

Evaluation

  • Auto-detects mode from the run directory name (pretrain_… vs finetune_…).
  • Pretrain eval renders reconstruction examples, error analysis, CLS-embedding t-SNE, plus a pretrain_metrics.txt file.
  • Finetune eval renders confusion matrix, regression scatter, t-SNE, plus finetune_metrics.txt.

Run discovery

  • list_runs.py walks runs/ and pretty-prints every run (size, status, model params, masking config, best metric). Supports --type, --latest, --details.

6. Modules & Classes

data/dataset.py

  • LIBSDataset — raw spectra wrapper.
  • MaskedLIBSDataset — masking (random / contiguous / variable / peak-biased) + BERT-style 80/10/10 replacement + get_masking_stats().
  • LabeledLIBSDataset — spectra + labels + optional concentrations.
  • MaskedLineTokenDataset, LineTokenLabeledDataset, LineFeaturesStore — runtime line-token path.
  • LineTokensDataset, MaskedLineTokensDataset, LineTokensLabeledDataset — pre-baked line_tokens_*.h5.
  • libs_worker_init_fn — re-seeds the per-worker numpy RNG so each epoch / worker gets a distinct, reproducible mask stream.

line_dictionary · line_features · line_tokenization

  • Theoretical line dictionary over Te×Ne → line_dict_*.h5.
  • Per-spectrum Voigt fits → line_features_*.h5 (6 channels).
  • build_line_tokens_cache → line_tokens_*.h5 (14 raw features + stats).
  • prepare_line_token_assets / prepare_line_tokens_assets in line_embedding_pipeline.py.

data/libs_pipeline.py

  • load_wavelength, load_sample_types, create_spectra — physics-based Voigt synthesis with an SQLite atomic-line DB.
  • SyntheticLIBSDataset — materialises spectra, caches them to HDF5 keyed by an md5 of the generation config.
  • extract_finetune_labels, cluster_compositions, get_or_make_splits — downstream-label helpers (concentration matrix, K-means cluster labels, deterministic train/val/test splits saved beside the cache).
  • build_dataset_from_config — top-level entry from libs_data.yaml.

models/libs_transformer.py

  • SpectralEmbedding — bin intensity MLP + mask token + PE.
  • LineTokenEmbedding — runtime quantum + element/ion Embeddings + Voigt → MLP.
  • LinearLineTokenEmbedding — z-score + nn.Linear(14, d_model) on pre-baked tokens.
  • TransformerEncoderBlock — pre-norm MHA + FFN with residual connections.
  • LIBSTransformer — stacked encoder, final norm, MIP head, helpers (get_embeddings, compute_mip_loss, freeze_encoder, num_parameters).

models/heads.py

  • ClassificationHead, RegressionHead, MaskedPredictionHead, MultiTaskHead.
  • BinnedQuantificationHead — one MLP branch per element producing n_bins logits.
  • concentration_to_bin / bin_to_concentration — discretisation utilities.
  • LIBSModelWithHeads — packaged encoder + heads.

training/pretrain.py

  • LIBSPretrainModule — Lightning module: AdamW, cosine + linear-warmup LR schedule, MSE on all 15 % of masked bins (mask-embedding applied only to the 80 % "mask token" positions).
  • PretrainDataModule — wraps MaskedLIBSDataset; logs masking statistics (avg masked bins, peak coverage, …).

training/finetune.py

  • LIBSFinetuneModule — task-dispatched losses (CE / MSE / per-element CE), pool = {cls, mean, cls_mean}, two AdamW param-groups (encoder LR ×0.1, head LR ×1).
  • FinetuneDataModule — wraps LabeledLIBSDataset.
  • Task-specific monitor: val/accuracy (cls), val/reg_mae (quant), val/bin_accuracy (binned), or val/loss.

utils/run_manager.py

  • RunManager — creates timestamped run dir, copies the source config, exposes {run_dir, checkpoint_dir, log_dir, evaluation_dir}, persists run_info.yaml, picks the best checkpoint for evaluation via get_checkpoint_for_mode.
  • list_runs, find_latest_run.

utils/metrics.py

  • Numeric: compute_classification_metrics, compute_regression_metrics, compute_mip_metrics, format_metrics.
  • Plots: plot_confusion_matrix, plot_spectra_comparison, plot_embeddings_tsne, plot_regression_scatter.

models/positional_encoding.py

  • SinusoidalPositionalEncoding — fixed sin/cos encoding, added element-wise after intensity projection.

9. Run Management

utils/run_manager.RunManager is the single source of truth for the on-disk layout of every experiment. It:

  • creates runs/{type}_{timestamp}_{experiment_name}/ at construction time;
  • copies the source YAML into config.yaml for reproducibility;
  • persists metadata into run_info.yaml via save_run_info() — written twice (status=running at start, status=completed at end, with best metric);
  • resolves the best checkpoint for evaluation: get_checkpoint_for_mode("pretrain") prefers final_model.pt → model_latest.pt → last.ckpt; get_checkpoint_for_mode("finetune") prefers last.ckpt (encoder + heads) → final_encoder.pt.

13. Glossary

MIP
Masked Intensity Prediction — the self-supervised objective: predict the original intensity at masked positions from the rest of the spectrum.
BERT 80/10/10
Of the masked positions: 80 % become a learnable mask token, 10 % a random intensity, 10 % are left unchanged. Prevents the model from learning a shortcut on the mask token alone.
Peak-biased masking
A configurable fraction (peak_bias_ratio) of the masked bins is forced to overlap with peak regions detected via a simple intensity threshold + ±5-bin shoulder expansion.
Run
A single training execution; lives in runs/{type}_{timestamp}_{experiment_name}/; contains a copy of the config, all checkpoints, logs, evaluation outputs and a run_info.yaml metadata file.
libs_data pipeline
Physics-based Voigt synthesis via --libs_data_config; overrides n_bins, n_classes, n_elements, and n_concentration_bins at runtime.
Line-token embedding
One token per retained line; line_token (runtime assembly) or line_token_linear (pre-baked HDF5).
Line tokens cache
line_tokens_{hash}.h5: 14-feature tensor per line per spectrum; built by data/line_tokenization.py.
Separated tokenization
Voigt fitting and feature merge happen offline; training loads line_tokens_*.h5 only (like synthetic_cache_*.h5 for spectra).
Line dictionary
Theoretical lines selected on a Te×Ne grid (default: top 10% per element, keep all if fewer than 10 lines; legacy intensity_threshold mode available) (line_dict_*.h5).
Pool
How encoder outputs feed downstream heads: cls (CLS token only), mean (mean over bins), or cls_mean (concatenation of both — head input dim is 2·d_model).
