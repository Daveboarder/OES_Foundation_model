# LIBS Foundation Model — Project Documentation

## 1. Goal

Build a **self-supervised foundation model for Laser-Induced Breakdown Spectroscopy (LIBS)**.  
The core idea: learn general spectral representations from *unlabeled* LIBS spectra using a masked prediction pretext task, then transfer those representations to *labeled* downstream tasks (material classification, element concentration regression) with fewer labeled samples.

---

## 2. Selected Architecture

**Transformer encoder** with a BERT-style self-supervised objective called **Masked Intensity Prediction (MIP)**.

| Component | Detail |
|---|---|
| Backbone | Custom Transformer encoder (pre-LayerNorm) |
| Embedding | Per-bin intensity projection `Linear(1 → d/2) → GELU → Linear(d/2 → d)`, plus learnable CLS token, learnable MASK token, sinusoidal positional encoding, and LayerNorm |
| Encoder block | Pre-norm multi-head self-attention + pre-norm FFN (GELU), residual connections |
| Pre-training head | MIP head: `Linear(d→d) → GELU → LayerNorm → Linear(d→1)` — predicts original intensity at masked positions |
| Downstream heads | `ClassificationHead` (class logits), `RegressionHead` (sigmoid output for concentrations) |
| Default config | d_model=256, n_heads=8, n_layers=6, d_ff=1024, ~3M params |
| Weight init | Xavier/Glorot for Linear, ones/zeros for LayerNorm |

---

## 3. Data

All data is **synthetic**, generated by `SyntheticLIBSGenerator`. There are **5 material classes**:

| ID | Material | Key peak positions (bin indices) |
|----|----------|----------------------------------|
| 0 | Iron | 300, 320, 520, 545, 850, 880 |
| 1 | Copper | 200, 230, 680, 720, 1200, 1250 |
| 2 | Aluminum | 400, 430, 900, 940, 1500, 1540 |
| 3 | Calcium | 600, 635, 1100, 1140, 1700, 1750 |
| 4 | Mixed | 300, 680, 900, 1100, 1800, 1900 |

Each spectrum is a 2048-bin vector (normalized to [0, 1]) composed of:
- Gaussian emission peaks at class-specific positions (with ±30% intensity variation and ±2 bin position jitter)
- Smooth polynomial baseline continuum
- Shot noise (proportional to √intensity) + Gaussian noise (σ=0.02)
- ~20% of dataset samples are mixtures of 2–3 materials with Dirichlet-distributed concentrations

## 4. Inputs and Outputs

### 4.1 Pre-training (self-supervised)

**Input to the dataset (`MaskedLIBSDataset`)**:
- Raw spectrum: `float32 [2048]`

**Output from the dataset** (per sample dict):
- `input`: masked spectrum `[2048]` — positions replaced per BERT-style rules (80% zeroed/mask-token, 10% random, 10% unchanged)
- `target`: original unmasked spectrum `[2048]`
- `mask`: boolean mask `[2048]` — True at masked positions
- `mask_type`: int mask `[2048]` — 0=unmasked, 1=mask-token, 2=random, 3=unchanged

**Input to the model** during pre-training:
- `input` (masked spectrum) and `mask` (boolean) are fed into `LIBSTransformer`

**Outputs from the model** during pre-training:
- `cls_embedding`: `[batch, 256]` — CLS token representation (not used during pre-training itself, but available)
- `sequence_output`: `[batch, 2049, 256]` — full hidden states including CLS
- `sequence_embeddings`: `[batch, 2048, 256]` — per-bin hidden states (excluding CLS)
- `mip_predictions`: `[batch, 2048]` — predicted intensity at every bin position

**Loss**: MSE computed only over masked positions between `mip_predictions` and `target`.

### 4.2 Fine-tuning (supervised downstream)

**Input to the dataset (`LabeledLIBSDataset`)**:
- Spectrum, class label, and concentration vector

**Output from the dataset** (per sample dict):
- `spectrum`: `float32 [2048]`
- `label`: `int64` scalar (0–4, dominant material class)
- `concentrations`: `float32 [5]` (per-element concentration weights, sum to 1)

**Input to the model** during fine-tuning:
- `spectrum` is fed into the encoder (no masking)

**Outputs from the model** during fine-tuning:
- `cls_embedding`: `[batch, 256]` — used by both heads below
- `class_logits`: `[batch, 5]` — raw logits for 5-class classification (from `ClassificationHead`)
- `concentrations`: `[batch, 5]` — predicted element concentrations in [0,1] via sigmoid (from `RegressionHead`)

**Losses**:
- Classification: CrossEntropyLoss on `class_logits` vs `label`
- Regression: MSELoss on predicted `concentrations` vs true `concentrations`
- When task=`both`: sum of both losses

### 4.3 Evaluation outputs

Pre-training evaluation produces:
- Reconstruction plots (original vs predicted spectra)
- Error analysis plots (error distribution, error vs intensity, predicted vs true scatter)
- t-SNE visualization of CLS embeddings (colored by material class)
- Metrics text file: MSE, RMSE, MAE, R²

Fine-tuning evaluation produces:
- Confusion matrix and per-class accuracy bar chart
- Per-element regression scatter plots
- t-SNE visualization of CLS embeddings
- Metrics text file: accuracy, balanced accuracy, per-class precision/recall/F1, regression MSE/MAE/R²

## 5. Masking Strategies (Pre-training)

The `MaskedLIBSDataset` supports multiple masking modes controlled by config:

| Strategy | Description |
|----------|-------------|
| **Random** | 15% of bins selected uniformly at random |
| **Contiguous blocks** | Blocks of variable size (default [25, 50, 100] bins) placed randomly until ~15% coverage |
| **Peak-biased** | 50% of masked bins forced to overlap with peak regions (detected via threshold > 0.2, expanded ±5 bins) |

BERT-style token replacement is applied to all strategies:
- 80% of masked bins → set to 0 (mask token replacement in input)
- 10% → replaced with random value in [0, 1]
- 10% → kept unchanged

Default config uses contiguous masking with peak bias enabled.

## 6. Training Pipeline

### Pre-training (`train_pretrain.py` → `training/pretrain.py`)

1. Load YAML config (Hydra-style but manual)
2. Create run directory via `RunManager`
3. Generate synthetic spectra (50k train, 5k val by default)
4. Build `LIBSTransformer` and wrap in `LIBSPretrainModule` (PyTorch Lightning)
5. Build `PretrainDataModule` with masking config
6. Train with Lightning Trainer: AdamW optimizer, cosine LR schedule with linear warmup (10 epochs warmup, 100 total epochs), bf16-mixed precision
7. Save best checkpoint (by val/loss) and `final_model.pt`

### Fine-tuning (`train_finetune.py` → `training/finetune.py`)

1. Load pretrained encoder from run directory or checkpoint
2. Create new run directory
3. Generate labeled synthetic data (10k samples)
4. Build `LIBSFinetuneModule` with encoder + task heads
5. Differential learning rates: encoder gets 0.1× the head LR
6. Optional encoder freezing
7. Train with Lightning, cosine LR with warmup
8. Run test evaluation, save `final_encoder.pt`

### Evaluation (`evaluate_model.py`)

Loads checkpoints and runs inference to produce plots and metrics files for both pretrain and finetune runs.

## 7. File-by-File Summary

### Entry Points

| File | Purpose |
|------|---------|
| `train_pretrain.py` | CLI entry point for pre-training. Parses args, loads config, generates data, creates model + Lightning module, runs training, saves model. |
| `train_finetune.py` | CLI entry point for fine-tuning. Loads pretrained encoder, attaches task heads, trains on labeled data, evaluates on test set. |
| `evaluate_model.py` | CLI entry point for evaluation. Loads checkpoints, runs pretrain/finetune evaluation, generates plots and metrics files. |
| `list_runs.py` | Utility to list and inspect experiment runs. |

### `models/`

| File | Contents |
|------|----------|
| `libs_transformer.py` | `SpectralEmbedding` (intensity projection + CLS/MASK tokens + positional encoding), `TransformerEncoderBlock` (pre-norm attention + FFN), `LIBSTransformer` (full encoder + MIP head). Core model. |
| `positional_encoding.py` | `SinusoidalPositionalEncoding` (fixed sin/cos, used by default), `LearnablePositionalEncoding`, `WavelengthEncoding` (alternatives, not used by default). |
| `heads.py` | `ClassificationHead` (Linear→GELU→Dropout→Linear), `RegressionHead` (same + sigmoid), `MaskedPredictionHead`, `MultiTaskHead`, `LIBSModelWithHeads`. Task-specific output heads. |

### `data/`

| File | Contents |
|------|----------|
| `synthetic_generator.py` | `SyntheticLIBSGenerator`: generates synthetic LIBS spectra with 5 material classes, Gaussian peaks, baseline, noise, and optional mixing. Produces (spectra, labels, concentrations). |
| `dataset.py` | `LIBSDataset` (base), `MaskedLIBSDataset` (masking for pre-training: random/contiguous/peak-biased + BERT-style replacement), `LabeledLIBSDataset` (labeled for fine-tuning). Also has `create_data_loaders` and `create_labeled_data_loaders` factory functions. |

### `training/`

| File | Contents |
|------|----------|
| `pretrain.py` | `LIBSPretrainModule` (Lightning module: MIP loss on masked positions, MSE, AdamW + cosine LR with warmup), `PretrainDataModule` (wraps `MaskedLIBSDataset`, logs masking stats). |
| `finetune.py` | `LIBSFinetuneModule` (Lightning module: CE + MSE losses, differential LR for encoder vs heads, optional encoder freezing), `FinetuneDataModule` (wraps `LabeledLIBSDataset`). |

### `utils/`

| File | Contents |
|------|----------|
| `run_manager.py` | `RunManager`: creates timestamped run directories, saves configs and metadata. `list_runs`, `find_latest_run` helpers. |
| `metrics.py` | `compute_classification_metrics`, `compute_regression_metrics`, `compute_mip_metrics`, `format_metrics`, and `plot_*` helpers for generating evaluation figures. |

### `config/`

| File | Purpose |
|------|---------|
| `config.yaml` | Standard config: 50k samples, d_model=256, 6 layers, 100 epochs, batch 64 |
| `config_local.yaml` | Local/debug config: 5k samples, smaller batch, 16GB GPU |
| `config_a100.yaml` | Large-scale: 500k samples, d_model=1024, 24 layers, ~500M params |

## 9. Summary of All Inputs and Outputs

### Model Inputs

| Phase | Tensor | Shape | Description |
|-------|--------|-------|-------------|
| Pre-train | masked spectrum | `[batch, 2048]` | Spectrum with ~15% of bins corrupted |
| Pre-train | mask | `[batch, 2048]` | Boolean: True at corrupted positions |
| Fine-tune | spectrum | `[batch, 2048]` | Clean spectrum (no masking) |

### Model Outputs

| Phase | Tensor | Shape | Description |
|-------|--------|-------|-------------|
| Pre-train | `mip_predictions` | `[batch, 2048]` | Predicted intensity for every bin (loss computed only at masked bins) |
| Pre-train | `cls_embedding` | `[batch, 256]` | CLS token embedding (not used for loss, but available for probing) |
| Pre-train | `sequence_embeddings` | `[batch, 2048, 256]` | Per-bin embeddings (MIP head reads from these) |
| Fine-tune | `class_logits` | `[batch, 5]` | Raw logits for 5-class material classification |
| Fine-tune | `concentrations` | `[batch, 5]` | Predicted per-element concentrations in [0, 1] (sigmoid) |
| Fine-tune | `cls_embedding` | `[batch, 256]` | CLS embedding used by both heads |
